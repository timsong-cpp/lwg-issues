Discussion:

This issue resulted out of this editorial change request.

In 22.2.4.1 [container.node.overview], there is Table 79 [tab:container.node.compat] which indicates which containers have compatible nodes.

It appears that rows along 

multimap<K, T, C1, A>     multimap<K, T, C2, A>

(i.e. multimaps with differing Compare functions) are missing from the table.

(Same for multiset and unordered_multiset/map.)

However, the introductory sentences in 22.2.4.1 [container.node.overview] do not relate "compatible nodes" with Table 79 and do not clearly state transitivity:

A node handle is an object that accepts ownership of a single element from an associative container (22.2.6 [associative.reqmts]) or an unordered associative container (22.2.7 [unord.req]). It may be used to transfer that ownership to another container with compatible nodes. Containers with compatible nodes have the same node handle type. Elements may be transferred in either direction between container types in the same row of Table 79 [tab:container.node.compat].
