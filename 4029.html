4029. basic_string accidentally fails to meet the reversible container requirements

Section: 23.4.3.1 [basic.string.general] Status: New Submitter: Jan Schultke Opened: 2023-12-18 Last modified: 2023-12-22 16:45:15 UTC

Priority: Not Prioritized

View all other issues in [basic.string.general].

View all issues with New status.

Discussion:

The complexity requirements for a reversible container (24.2.2.3 [container.rev.reqmts]) are that each function must have constant complexity. The corresponding member functions in 23.4.3.4 [string.iterators] have no complexity requirements, and basic_string unintentionally is not a reversible container (unless the implementation coincidentally provides constant complexity member functions).

Proposed resolution:

This wording is relative to N4971.

[Drafting Note: The proposed wording is similar to the specification in 24.3.11.1 [vector.overview] p2 and suggests to simply strike 23.4.3.4 [string.iterators] because it doesn't say anything new compared to 24.2.2.2 [container.reqmts] and 24.2.2.3 [container.rev.reqmts].

Alternatively, one could add a

Complexity: Constant.

paragraph to each function in 23.4.3.4 [string.iterators], but that would be less clearer and would not explicitly say basic_string meets the reversible container requirements. ]
