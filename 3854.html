3854. §[res.on.exception.handling]/3 should not be applied to all standard library types

Section: 16.4.6.14 [res.on.exception.handling] Status: New Submitter: Jiang An Opened: 2023-01-07 Last modified: 2023-02-01

Priority: 3

Discussion:

Some standard library types, such as std::pair, std::tuple, and std::array, are currently allowed to have a member or element type with a destructor that is noexcept(false). In order to conform to 16.4.6.14 [res.on.exception.handling]/3, these types can't always have implicitly declared destructors because the implicit exception specification may be noexcept(false).

However, it may be intentional to implement these types with implicitly declared destructors, and mainstream implementations (libc++, libstdc++, and MSVC STL) are already doing so.

Furtherly, explicitly declared destructors are not allowed for algorithm result types (26.5 [algorithms.results]), while currently these class templates are allowed to be instantiated with classes with potentially throwing destructors, which is contradictory to 16.4.6.14 [res.on.exception.handling]/3.
