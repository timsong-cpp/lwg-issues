3305. any_cast<void>

Section: 20.8.4 [any.nonmembers] Status: New Submitter: John Shaw Opened: 2019-10-16 Last modified: 2019-11-04 13:23:43 UTC

Priority: 2

View all other issues in [any.nonmembers].

View all issues with New status.

Discussion:

any foo;
void* p = any_cast<void>(&foo);

Per 20.8.4 [any.nonmembers]/9, since the operand isn't nullptr and operand->type() == typeid(T) (because T = void in this case), we should return a pointer to the object contained by operand. But there is no such object.

We need to handle the T = void case, probably by just explicitly returning nullptr.

[2019-11 Priority to 2 during Monday issue prioritization in Belfast. There is implementation divergence here.]

Proposed resolution:

This wording is relative to N4835.

  1. Modify 20.8.4 [any.nonmembers] as indicated:

    template<class T>
  const T* any_cast(const any* operand) noexcept;
template<class T>
  T* any_cast(any* operand) noexcept;

    -9- Returns: If operand != nullptr && operand->type() == typeid(T) && is_object_v<T>, a pointer to the object contained by operand; otherwise, nullptr.
