2582. §[res.on.functions]/2's prohibition against incomplete types shouldn't apply to type traits

Section: 20.15 [meta] Status: WP Submitter: Tim Song Opened: 2016-01-07 Last modified: 2016-03-06

Priority: 0

View other active issues in [meta].

View all other issues in [meta].

Discussion:

17.5.4.8 [res.on.functions]/2.5 says that the behavior is undefined "if an incomplete type is used as a template argument when instantiating a template component, unless specifically allowed for that component."

This rule should not apply to type traits — a literal application would make is_same<void, void> undefined behavior, since nothing in 20.15 [meta] (or elsewhere) "specifically allows" instantiating is_same with incomplete types.

Traits that require complete types are already explicitly specified as such, so the proposed wording below simply negates 17.5.4.8 [res.on.functions]/2.5 for 20.15 [meta].

    -?- Unless otherwise specified, an incomplete type may be used to instantiate a template in this subclause.
