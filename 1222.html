1222. condition_variable incorrect effects for exception safety

Section: 30.5 [thread.condition] Status: C++11 Submitter: Jeffrey Yasskin Opened: 2009-09-30 Last modified: 2016-01-28 10:19:27 UTC

Priority: Not Prioritized

View all other issues in [thread.condition].

Discussion:

30.5.3 [thread.condition.condvar] says: 

void wait(unique_lock<mutex>& lock);

...

Effects:

  • ...
  • If the function exits via an exception, lock.unlock() shall be called prior to exiting the function scope.

Should that be lock.lock()?

[ 2009-11-17 Moved to Tentatively Ready after 5 positive votes on c++std-lib. ]

Proposed resolution:

Change 30.5.3 [thread.condition.condvar] p10: 

void wait(unique_lock<mutex>& lock);

...

Effects:

  • ...
  • If the function exits via an exception, lock.unlock() shall be called prior to exiting the function scope.

And make a similar change in p16, and in 30.5.4 [thread.condition.condvarany], p8 and p13.
