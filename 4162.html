4162. Worst time complexity of non-parallel versions of nth_element is underspecified

Section: 26.8.3 [alg.nth.element] Status: New Submitter: Jiang An Opened: 2024-09-29 Last modified: 2024-10-09

Priority: 3

View all other issues in [alg.nth.element].

View all issues with New status.

Discussion:

Currently, 26.8.3 [alg.nth.element] doesn't specify the worst time complexity for nth_element without ExecutionPolicy parameter, which seemingly allows a complexity that is 𝒪(N2) or even worse. Presumably we should make the worst time complexity consistent between parallel and non-parallel versions. For sort, LWG 713 already strengthened complexity requirements.

[2024-10-09; Reflector poll]

Set priority to 3 after reflector poll.

"This seems to require changes to implementations for them to meet this complexity guarantee."
